# 报告生成机制

## 输入
- `EvaluationResult`
- `PositionSnapshot`
- `PnLSnapshot`
- `EvidencePacket`
- `WindowDecision`
- 用户今日关注点

## 输出
- `DailyReviewReport`

## 结构模板（强约束）
1. 今日结论
2. 关键事实
3. 你看对了什么
4. 你看错了什么
5. 可能是超前洞察
6. 执行/仓位/风控评估
7. 今日强化规则
8. 今日风险预警
9. 下一步观察清单
10. 策略修正建议（可选）

## 生成规则
- 证据优先：每条评价要可追溯到 evidence id
- 奖惩对称：必须输出“正确洞察”与“偏差点”
- 时序审慎：短期未兑现不等于错误
- 建议可执行：建议必须可观测、可检查

## 失败处理
- 若证据完整度不足：报告明确标注“暂判/待验证”
- 若持仓数据缺失：执行和仓位结论降级并给出补充请求
